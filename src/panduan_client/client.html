Gambar diatas adalah contoh error ini ketika koneksi internet terputus saat automatic chunk berlangsung, error ini bersifat recoverable tapi mengakibatkan waktu proses menjadi lebih lama.

Solusi dari error ini adalah mencari komputer client yang memiliki memori RAM yang lebih besar untuk memproses data yang diinputkan, apabila memori RAM client sudah cukup besar dan masih mendapat error ini maka tambah jumlah host server (komputer) yang terkoneksi pada sistem.

Berikut adalah pembahasan tiga fungsi tombol hasil proses.

      1. Tombol "Buka Stat" mencari file pada folder Statistik Log dengan nama file "Log <konfigurasi algorimtma>(<parameter>)", dimana apabila terdapat nama file yang sama akan ditampilkan file dengan nilai increment tertinggi ("Log <konfigurasi algorimtma>(<parameter>)(<increment>)"). File yang terpilih kemudian akan ditampilkan melalui teks editor bawaan yang dipilih oleh sistem operasi pengguna. isi dari file adalah waktu proses terhadap masing - masing file sejumlah parameter running test. Gambar diatas adalah contoh dari pembukaan file statistik ketika pengguna menekan tombol ini.

      2. Tombol "Buka Hasil Sort" berfungsi menampilkan file hasil pengurutan, pengguna harus memilih terlebih dahulu file yang ingin diurutkan. dimana direktori file akan berbeda - beda berdasarkan konfigurasi algoritma. untuk konfigurasi sequential merge sort file terdapat pada folder "outputfile/Sequential MergeSort", untuk konfigurasi parallel merge sort file terdapat pada folder "outputfile/Parallel MergeSort", untuk konfigurasi distributed merge sort file terdapat pada folder "outputfile/Distributed MergeSort". Pada umumnya pengguna tidak perlu khawatir tentang direktori file hasil pengurutan karena akan dibuka otomatis saat menekan tombol ini. Hal yang harus pengguna perhatikan adalah file yang ingin dibuka, sebab nama file bisa duplikasi dan apabila ada nama file sumber dan konfigurasi parameter yang sama akan ditambah increment. Format file hasil pengurutan adalah : "<nama file sumber > <ukuran array> <parameter>", sementara apabila terdapat duplikasi nama file maka sesi baru akan membentuk file baru dengan increment berformat : "<nama file sumber > <ukuran array> <parameter> (<increment>)". hasil pengurutan adalah file yang memiliki nama sama dengan file input. File terpilih akan otomatis dibuka menggunakan aplikasi text editor bawaan sistem operasi pengguna. Gambar diatas menunjukkan fungsi tombol "Buka Hasil Sort".

      3. Tombol "Buka Log" berfungsi untuk membuka file rekaman riwayat log pada folder "Log Client (autosave)". File yang dicari adalah file bernama "autosave.txt", apabila terdapat duplikasi maka aplikasi akan membuat file baru setiap sesi aplikasi secara increment, jadi format file menjadi "autosave(<increment>).txt". umumnya pengguna tidak perlu mengkhawatirkan penamaan file. Sebab ketika pengguna menekan tombol ini file akan otomatis dibuka pada aplikasi text editor sistem operasi pengguna tanpa pengguna harus memilih file lagi. Tombol ini hanya akan berfungsi jika pengguna sudah pernah mencentang checkbox "Auto Save Log" pada sesi yang sama, jika tidak akan muncul error seperti gambar diatas. Adapun konsep "Auto Save Log" adalah menyimpan riwayat proses hanya pada saat ketika checkbox sudah dicentang. Apabila checkbox dicentang sesudah proses tertentu maka proses sebelumnya tidak akan direkam kedalam file. Gambar Berikut mendemonstrasikan fungsi dari checkbox "Auto Save Log".

        4. Dapat dilihat dari gambar diatas bahwa perekaman riwayat proses hanya dimulai ketika centang sudah dibuat sebelum proses dimulai.

      4. Tombol "Restart" berfungsi untuk mereset tampilan kembali ke tahap awal. Gambar diatas mendemonstrasikan fungsi tombol restart.

Catatan :
  1. Sebaiknya dalam satu host (komputer) hanya membuka satu aplikasi client, karena akan menyebabkan interferensi listening socket kedua aplikasi client yang dibuka bersamaan.
  2. Apabila tampilan tidak responsif, aplikasi harus dimatikan paksa secara manual (bisa lewat cmd) atau process manager.
  3. Ketika menekan tombol start apabila tombol masih kelap - kelip RGB atau progress bar belum muncul, berarti proses masih belum dimuulai dan pengguna harus menekan tombol start lagi untuk memulai proses.
  4. Sebaiknya dalam satu host (komputer) hanya membuka satu aplikasi client atau server, tidak boleh kedua aplikasi dibuka bersamaan kecuali untuk "testing". Apabila kedua aplikasi dibuka bersamaan maka host server pada komputer lain akan sulit dikendalikan sebab sistem dapat mengambil network interface yang salah (localhost) yang menyebabkan loopback calling dan mengabaikan packet dari host pada komputer lain.
